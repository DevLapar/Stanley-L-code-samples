** English below **
Als programmeur codeer ik volgens het OOP principe waar het draait om object georienteerd te werk te gaan. Ik volg het MVCS (Model, View, Controller, Services) structuur waarvan ik vind dat het een overzichtelijke manier van file & werk structuur is. Dat logica van templates (views, zoals HTML) gescheiden wordt vind ik persoonlijk erg prettig omdat de combinatie van PHP code met HTML snel onoverzichtelijk kan worden. Dat is voor mij ook de reden om HTML en de daarbij horende aspecten zoals JS/CSS etc puur via front-end te coderen met oa het behulp van een framework. PHP wordt door mij dus voornamelijk gebruikt als REST api waar ik de database en de desbetreffende models kan aanpassen. En ook de wat zwaardere taken zal uitvoeren wat mogelijk te veel memory kan vreten client-side.

Alle code dat ik in deze repo deel is voor demonstratie doeleinden om aan te tonen aan partijen wat mijn skills zijn, mag er interesse zijn. De gedeelde code is afkomstig uit persoonlijke projecten en producten (die oa. ook online zijn), en weergeeft hoe ik te werk ga. De gedeelde code in deze folder en alle andere folders zijn bij deze publiek gemaakt en mogen gedeelt worden met 3e partijen waarvan er interesse is.

Omdat ik dagelijks voornamelijk in het engels werk en er ook mogelijk interesse is vanuit internationale (remote) bedrijven zullen de rest van de READMEs in het engels worden geschreven. Daarnaast zijn comments in mijn code in het engels geschreven.




** English **
As a programmer I respect the OOP principal thus I write my code with object oriented programming in mind. I adhere the MVCS programming structure as I believe it defines a clear construction of how code should be seperated from logic to views. Although MVCS implies that also Views should be present, I have seperated that by using a Front-end framework and purely write HTML (along with JS/CSS etc) frontend. That means I mainly use PHP as a REST Api to modify, access, and perform the heavy duty operations on the server side.

The code that I have shared in this repo are purely for demonstrational purposes to back my expertise in provided languages, for anyone that is interested. Code of this repo are pieces from several personal projects and products (which are also live) and represent my working ethics. Code shared in this folder and all respective child folders & files are hereby made public are may be used freely or shared with designated parties.
